Explore how to control element layout with CSS, including the display property and positioning methods.

4. CSS Layout Techniques

CSS provides various techniques for controlling the layout and positioning of elements on a web page. Mastering these techniques is crucial for creating responsive and visually appealing web designs.

4.1. Display Property

The display property defines how an element is displayed on the web page. There are several common values for the display property:

  • Block: The element takes up the full width available and starts on a new line. Examples include <div>, <p>, and <h1>.
  • Inline: The element only takes up as much width as necessary, and it flows within the content. Examples include <span> and <a>.
  • Inline-Block: Similar to inline, but the element can have width and height applied, while still flowing inline.

• Block

In this example, the div elements are displayed as block elements, meaning they take up the full width of their container and appear one after the other vertically.

div {
    display: block;
}

• Inline

Inline elements, such as <span>, only take up as much space as their content requires. They flow inline with other elements, meaning they can be placed side by side in the same line.

span {
    display: inline;
}

• Inline-Block

The inline-block display type combines aspects of both block and inline elements. The element behaves like an inline element, allowing it to sit next to other elements on the same line, but it also accepts width and height properties like a block element.

img {
    display: inline-block;
}

4.2. Positioning Elements

CSS provides different methods for positioning elements on the web page. By manipulating the position property, you can control how elements are placed in relation to their normal position or the page as a whole.

  • Static: The default position. Elements are positioned according to the normal flow of the document.
  • Relative: The element is positioned relative to its normal position, allowing it to be moved without affecting other elements.
  • Absolute: The element is positioned relative to its closest positioned ancestor or the document if none exists.
  • Fixed: The element is positioned relative to the browser window and remains fixed in place when scrolling.

• Static

The static position is the default position of all HTML elements. They appear in the normal flow of the document without any special positioning.

p {
    position: static;
}

This element is in its default static position.

• Relative

Elements with position: relative are positioned relative to their original static position. You can offset their position using the top, right, bottom, or left properties.

.relative {
    position: relative;
    top: 10px;
    left: 20px;
}

This element is positioned 10px down and 20px to the right from its original position.

• Absolute

An element with position: absolute is removed from the normal document flow and is positioned relative to its nearest positioned ancestor (i.e., an ancestor with a position other than static). If no such ancestor exists, the element is positioned relative to the <html> document.

.absolute {
    position: absolute;
    top: 50px;
    left: 100px;
}
This element is positioned 50px from the top and 100px from the left.

• Fixed

With position: fixed, the element is positioned relative to the browser window, meaning it stays in the same place even when the user scrolls the page.

.fixed {
    position: fixed;
    top: 0;
    right: 0;
}
